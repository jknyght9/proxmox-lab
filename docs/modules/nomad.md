# Nomad Cluster

The Nomad module deploys a 3-node HashiCorp Nomad cluster for container orchestration. Each node functions as both a server and client, providing high availability and workload scheduling across the cluster.

## Overview

| Property | Value |
|----------|-------|
| **Module Path** | `terraform/vm-nomad` |
| **VM Count** | 3 (nomad01, nomad02, nomad03) |
| **VMID Range** | 905-907 |
| **Template** | VMID 9002 (Nomad template) |
| **Network** | vmbr0 (external) |
| **Storage** | GlusterFS replicated volume |

## Architecture

### Cluster Configuration

```mermaid
graph TB
    subgraph cluster["Nomad Cluster (dc1, region: global)"]
        N1[nomad01<br/>Server+Client<br/>VMID 905]
        N2[nomad02<br/>Server+Client<br/>VMID 906]
        N3[nomad03<br/>Server+Client<br/>VMID 907]

        N1 <-->|Raft Consensus<br/>Port 4647| N2
        N2 <-->|Raft Consensus| N3
        N3 <-->|Raft Consensus| N1
    end

    subgraph storage["GlusterFS Replicated Volume"]
        GFS[/srv/gluster/nomad-data<br/>Replica 3]
    end

    N1 & N2 & N3 --> GFS

    subgraph jobs["Nomad Jobs"]
        TRAEFIK[Traefik]
        VAULT[Vault]
        AUTHENTIK[Authentik]
    end

    N1 --> jobs
```

### Key Features

- **Dual Role**: Each node is both server (for scheduling) and client (for running jobs)
- **High Availability**: 3-server Raft quorum survives single node failure
- **Shared Storage**: GlusterFS provides replicated storage for job data
- **DNS Discovery**: Servers find each other via DNS FQDNs

## Deployment

### Prerequisites

- Packer template built (VMID 9002)
- DNS cluster deployed and functioning
- step-ca deployed for TLS certificates
- `cluster-info.json` configured with network settings

### Terraform Variables

```hcl
# In terraform/terraform.tfvars

dns_postfix         = "mylab.lan"
dns_primary_ipv4    = "<dns-01-ip>"
network_interface_bridge = "vmbr0"
ssh_public_key_file = "../crypto/lab-deploy.pub"
vm_storage          = "local-lvm"  # or shared storage

# Node IP mapping (from cluster-info.json)
proxmox_node_ips = {
  pve01 = "10.1.50.2"
  pve02 = "10.1.50.3"
  # ...
}
```

### Cloud-Init Configuration

Each Nomad node is configured via cloud-init with:

- Hostname (nomad01, nomad02, nomad03)
- Static IP from user-defined range
- DNS resolution via primary DNS
- GlusterFS mount at `/srv/gluster/nomad-data`
- Nomad server and client configuration

Template location: `terraform/vm-nomad/cloudinit/nomad_cloudinit.yml`

## GlusterFS Storage

### Volume Configuration

```bash
# Volume details (configured automatically)
Volume Name: nomad-data
Type: Replicate
Volume ID: <auto-generated>
Status: Started
Number of Bricks: 3 (1 x 3 = 3)
Brick1: nomad01:/srv/gluster/nomad-data-brick
Brick2: nomad02:/srv/gluster/nomad-data-brick
Brick3: nomad03:/srv/gluster/nomad-data-brick
```

### Mount Point

- **Path**: `/srv/gluster/nomad-data`
- **Mount Type**: GlusterFS native client
- **Permissions**: 755 (root:root)
- **Auto-mount**: Yes (via `/etc/fstab`)

### Usage by Jobs

Jobs mount subdirectories for persistent storage:

```hcl
# Example from vault.nomad.hcl
volumes = [
  "/srv/gluster/nomad-data/vault:/data/vault"
]
```

Directory structure:
```
/srv/gluster/nomad-data/
├── vault/          # Vault storage backend
├── authentik/      # Authentik data
│   ├── postgres/   # PostgreSQL data
│   ├── redis/      # Redis data
│   ├── media/      # Uploaded files
│   └── certs/      # Certificates
├── traefik/        # Traefik config and certs
└── certs/          # Shared CA certificates
```

## Nomad Configuration

### Server Configuration

```hcl
# Generated in cloud-init
server {
  enabled          = true
  bootstrap_expect = 3

  server_join {
    retry_join = [
      "nomad01.mylab.lan",
      "nomad02.mylab.lan",
      "nomad03.mylab.lan"
    ]
  }
}

datacenter = "dc1"
region     = "global"
```

### Client Configuration

```hcl
client {
  enabled = true

  options {
    "driver.docker.enable" = "1"
  }
}
```

### Port Mapping

| Port | Protocol | Purpose |
|------|----------|---------|
| 4646 | TCP | HTTP API and Web UI |
| 4647 | TCP | RPC (server communication) |
| 4648 | TCP/UDP | Serf gossip |

## Operations

### Accessing the Cluster

```bash
# SSH to any node
ssh ubuntu@nomad01.mylab.lan

# View cluster members
nomad server members

# View node status
nomad node status

# View running jobs
nomad job status
```

### Web UI

Access Nomad UI at: `http://nomad01.mylab.lan:4646`

Features:
- Job status and history
- Allocation logs
- Node health monitoring
- Service registry

### Deploying Jobs

Jobs are deployed via `setup.sh` menu or manually:

```bash
# Via setup.sh
./setup.sh
# Select option 7, 8, or 9

# Manual deployment
nomad job run nomad/jobs/vault.nomad.hcl
```

### Checking Logs

```bash
# View logs for a specific job
nomad alloc logs -job vault

# Follow logs in real-time
nomad alloc logs -f -job authentik

# View logs for specific task
nomad alloc logs -job authentik -task server
```

## Job Constraints

All current jobs are pinned to nomad01 for consistency:

```hcl
constraint {
  attribute = "${attr.unique.hostname}"
  value     = "nomad01"
}
```

This ensures:
- Consistent DNS (all service DNS points to nomad01)
- ACME challenges work (single Traefik instance)
- No cross-node routing complexity

Future jobs can be scheduled across all nodes by removing this constraint.

## Vault Integration

Nomad integrates with Vault using Workload Identity Federation (WIF):

### Configuration

```hcl
# In Nomad server config
vault {
  enabled = true
  address = "http://127.0.0.1:8200"

  default_identity {
    aud  = ["vault.io"]
    ttl  = "1h"
  }
}
```

### Job Usage

```hcl
# In job specification
vault {
  role        = "authentik"
  change_mode = "restart"
}

template {
  data = <<EOH
{{ with secret "secret/data/authentik" }}
PASSWORD={{ .Data.data.postgres_password }}
{{ end }}
EOH
  destination = "secrets/app.env"
  env         = true
}
```

See [Vault Operations](../operations/vault-operations.md) for details.

## Troubleshooting

### Node Not Joining Cluster

```bash
# Check Nomad logs
sudo journalctl -u nomad -f

# Verify DNS resolution
nslookup nomad01.mylab.lan
nslookup nomad02.mylab.lan
nslookup nomad03.mylab.lan

# Check connectivity
nc -zv nomad01.mylab.lan 4647
```

### GlusterFS Issues

```bash
# Check if volume is mounted
df -h | grep nomad-data

# Remount if needed
sudo umount /srv/gluster/nomad-data
sudo mount -t glusterfs nomad01:/nomad-data /srv/gluster/nomad-data

# Check volume heal status
sudo gluster volume heal nomad-data info
```

### Job Stuck in Pending

```bash
# Check allocation status
nomad job status <job-name>
nomad alloc status <alloc-id>

# Check node eligibility
nomad node status -verbose

# Force evaluation
nomad job dispatch <job-name>
```

## Backup and Recovery

### Critical Data

- **GlusterFS data**: `/srv/gluster/nomad-data/`
- **Nomad state**: Stored in Raft log (automatic replication)
- **Job definitions**: `nomad/jobs/*.nomad.hcl` (version controlled)

### Backup Procedure

```bash
# Backup GlusterFS data from any node
sudo tar -czf nomad-data-backup.tar.gz -C /srv/gluster nomad-data/

# Export job definitions
nomad job inspect vault > vault-exported.json
```

### Recovery

```bash
# Restore GlusterFS data
sudo tar -xzf nomad-data-backup.tar.gz -C /srv/gluster/

# Redeploy jobs from definitions
nomad job run nomad/jobs/vault.nomad.hcl
```

## Security

### Access Control

- SSH access: Key-based authentication only
- Nomad API: No ACLs configured (local network only)
- Firewall: Ports 4646-4648 open between cluster nodes

### Secrets Management

- No secrets stored in job files
- All secrets fetched from Vault at runtime
- Workload Identity tokens expire after 1 hour
